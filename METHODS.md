# Methods and Validation Notes

This repository packages a local wildlife video-processing workflow built around:

- Frigate-style `.mp4` clip ingestion
- MegaDetector object detection
- DeepFaune New England species classification
- per-clip JSON reporting
- hourly summary JSON reporting
- retention cleanup and optional positive-clip copying

## Intended Use

The pipeline is intended for local or self-hosted wildlife monitoring workflows where clips are already being generated by a camera or NVR system. The primary goal is operational triage and species-level review support, not autonomous scientific claim generation.

## Processing Flow

For each selected clip, the pipeline:

1. Reads clip metadata and timing information.
2. Extracts frames at a configured interval.
3. Runs MegaDetector on the extracted frames.
4. Sends animal detections to DeepFaune New England for classification.
5. Writes a structured per-clip JSON report.
6. Optionally aggregates results into hourly summaries.

## Current Validation Scope

The current public validation state is limited and should be described honestly:

- Unit tests cover configuration loading, path validation, hourly selection logic, report generation, retention behavior, and demo-mode output generation.
- The pipeline has been exercised against archived Frigate-style clips on the maintainer's local system.
- The repository includes example outputs and a demo mode to make the reporting format inspectable without local models or a live camera environment.

## What Has Not Yet Been Established

The following have not yet been established in a rigorous scientific sense:

- species-level precision and recall across a representative field dataset
- performance under consistent night-time infrared conditions
- formal confusion analysis across target taxa
- cross-site generalization beyond the maintainer's current deployment context
- reproducibility across a broad range of hardware and camera configurations

## Interpretation Guidance

Outputs from this repository should be treated as machine-assisted classifications. They may be useful for screening, workflow automation, and follow-up review, but they should not be treated as definitive biological records without additional validation.

For scientific or management use, downstream review should ideally include:

- manual inspection of source clips or extracted frames
- site-specific validation against known species occurrences
- threshold and error-rate characterization for the deployment environment

## Practical Positioning

This project is best understood as:

- an open workflow implementation
- an integration layer between existing wildlife models and local video infrastructure
- a starting point for further validation, not a completed scientific benchmark

That is still a legitimate contribution. It makes the operational method inspectable, reusable, and easier for others to evaluate or extend.
